## Root causes, ranked by test impact (evidence = testng-results.xml exception clusters + job logs)

### 1. v1.48 SiteVisit flow remap missing at run time — ~182 fails + 119 skips
`SiteVisit_phase1`: 61 of 72 fails are ONE signature — "Failed to click element
after N attempts + coordinate fallback" (the WO-card/walkthrough entry the tests
click moved in v1.48). `SiteVisit_phase2` (50 fails): "Could not add ATS to
Quick Count" ×9, "requires at least one asset in the room for (long-press)
context menu" ×8, "Could not reach Quick Count screen" ×3, plus TC_JOB_100/101
IR/Visual-photo-filename 360s hangs → breaker (44 skips) → 240-min wall.
`SiteVisit_phase3` (60 fails): "Could not navigate to Existing Asset tab" ×13,
"…Photo Walkthrough screen" ×11, "…Photograph OCPD screen" ×10, "…What's Next?
screen" ×10 — precisely the WO→Photo-Walkthrough surface that `8090737`
remapped. **Largely already fixed on main** — this run predates it. Expect a
big SiteVisit recovery on the next full run; whatever remains (Quick Count /
room context menu) is the residual remap surface.

### 2. v1.48 Issue Details/creation DOM change — ~76 fails across Issue_Phase1/2/3
Signatures (fast 2–10s assertion failures, not hangs):
- Dropdown *reads* return empty: "Issue Class field should show 'NEC Violation'
  after selection (was: '')", same for NFPA/OSHA/priority.
- Option lists read as empty/short: "known NEC subcategory option … found 0/3",
  "expected ≥5 of 11 NEC options, found 0", "Status options Expected: 4,
  Actual: 1", 13× + 7× "Chapter N.N.N option should be present in the NFPA 70B
  subcategory list".
- Details controls "missing": Gallery button, Delete Issue button, "Open tab
  selected by default".
These are the same *family* as the SiteVisit remap: v1.48 changed the
sheet-picker/details DOM and the Issues page objects still read the old
structure. **This is the biggest UNFIXED cluster** → needs an Issues remap pass
from live v1.48 DOM evidence (like commit `8090737` did for SiteVisit).
The rerun reproduced all 43 Issue_Phase1 failures identically on a fresh
simulator, so it is deterministic, not flake.

### 3. Giant-DOM WDA wedges (known mechanism) — ~30 direct 360s-timeout fails, →breaker→ ~370 cascade skips
Six-minute `ThreadTimeoutException` hangs on the usual suspects, each wedging
WDA so the *next* tests died too, tripping the breaker and skipping the rest of
the job:
- Asset subtype/class-change pickers: MCC/MOT/REL/PB `*_AST_*` (assets-p6, 102
  skips), "Asset class change to 'Relay' exceeded 180s budget".
- Asset save/edit flows: ATS_ECR_31 (assets-p1 → 85 skips), TC_UPS_01/07
  (assets-p5 → 93 skips), CB_EAD_24 (assets-p2 → 96 skips).
- OSHA subcategory interactions (issues-p2: 6 hangs → 20 "Driver not
  initialized" skips).
- Offline UC1–UC4 all hung 360s (offline job: 0 passes, 72 skips — 34 of those
  are the by-design "WiFi toggle unavailable on simulator" precondition skips).
- Connections TC_CONN_002/004 hangs early, then job crawled and hit the
  240-min wall (59 skips).

### 4. Asset edit "save not confirmed" — 9 fails (assets-p2 CB_EAD_1x/2x) + repeats in rerun
`VerificationError: isAssetSavedAfterEdit: no positive save evidence … 'Save
Changes' is still visible after a real Save click — save likely failed
(validation error?)`. Deterministic across main run + rerun. Either a v1.48
save-flow change or a real app validation regression on Circuit Breaker edit —
needs one manual repro session on the CB asset to decide.

### 5. Mixed-class rerun suites start on a foreign screen — 27 of the 83 rerun fails
`navigateToAssetList: could not reach the Asset List. assetsTab=false,
sitePicker=false` across Asset_Phase1/2/3/4 in the rerun only. The previous
class died on a non-Dashboard screen and the next class inherited it. **Fixed**
by `RERUN_LOGIN_FIRST` (changelog 105).

### 6. Smaller clusters
- Location: 9× "Should navigate to New Floor screen" (location-p1) + New Room
  save/Done-button failures (location-p2, 26 fails) — v1.48 location form
  changes suspected, same remap treatment needed.
- Auth: 5× "Failed to click element after N attempts" on Continue (TC38 also
  cannot find any TextField on the user-data screen — v1.48 profile screen?).
- ZP-323: 18 fails — 17 of them are `navigateToAssetList: assetsTab=false,
  sitePicker=false`: the job's FIRST test (TC_ZP323_06_01, click-failure)
  left the app off-Dashboard and every later test aborted navigation. Same
  stranded-screen mechanism as the rerun's 27 — but inside a single-class
  main job. If it recurs on the next run, add `-DRERUN_LOGIN_FIRST=true` to
  the zp323 job's mvn flags (the flag is job-agnostic; ~1s/test fast path).
- Smoke job: driver never initialized (0 pass / 4 init-skips) — WDA warm-up
  passed but the Java session could not start; isolated to this runner.

## Skip taxonomy (the 704 main-run skips)

| Kind | Count (approx) | Meaning |
|---|---|---|
| Dead-session breaker OPEN | ~440 | Deliberate fast-skip after 5 consecutive driver-dead outcomes |
| Driver init failed / not initialized | ~40 | WDA could not give a session at setup |
| RunHealth wall-clock 240 min | ~170 (sitevisit p2/p3, connections) | Job budget exhausted before the class list finished |
| testSetup 6-min timeout | ~9 | Setup itself wedged |
| Genuine preconditions (WiFi toggle, camera, known app bugs) | ~40 | By design, will always skip on CI |

**Reading rule:** breaker/wall/init skips are *denied runs*, not test verdicts.
They are exactly what the (now sharded) fresh-sim rerun exists to retry.
